## 12. IP资产登记与NFT铸造模块最小改动补齐处理文档

### 12.1 目标与原则

**目标：**
- 在不推翻现有架构的前提下，补齐第3章关键缺口，达到可验收状态。

**最小改动原则：**
- 保留当前路由分层架构（`/nft/assets`、`/nft/minting`、`/nft/history`）。
- 优先复用已有能力，不新增重复页面。
- 优先新增组件/函数，不做大规模重命名与模型重构。
- 接口兼容优先：新增别名路径兼容规格，不破坏现有调用。

### 12.2 当前差距与补齐策略

| 编号 | 差距项 | 当前状态 | 最小改动补齐方案 | 优先级 |
|------|--------|----------|------------------|--------|
| G1 | 单个铸造确认Modal（资产摘要/钱包/版税/Gas）未在主流程统一启用 | 在 `pages/NFT/index.tsx` 有实现，但主路由主要使用 `pages/NFT/assets/index.tsx` | 抽取“单铸造确认弹窗”组件，接入 `pages/NFT/assets/index.tsx` 的单铸造入口 | P0 |
| G2 | 铸造前IPFS元数据JSON预览缺失 | 后端有元数据生成逻辑，前端未展示 | 在确认Modal增加“元数据预览”区域；由前端根据当前资产与附件构造预览结构 | P0 |
| G3 | 源码资产前端SHA-256计算按钮缺失 | 后端已支持哈希校验接口 | 在资产登记上传区新增“计算文件哈希”按钮，使用浏览器 `crypto.subtle.digest` 计算并展示；提交时仅提交哈希记录（源码类） | P0 |
| G4 | 上传后CID展示/进度不完整 | 目前以合并接口创建资产+附件 | 在上传区增加逐文件上传状态展示（处理中/成功/失败）与CID列表回显；不改后端核心流程 | P1 |
| G5 | API路径与规格不一致 | 现有路径可用，但与文档不完全一致 | 后端新增别名路径：`/ipfs/upload-file`、`/ipfs/upload-json`、`/nft/mint-status/{task_id}`、`/nft/retry-mint/{asset_id}`，内部复用现有处理函数 | P1 |
| G6 | 数据模型字段命名与规格存在差异 | 现有模型可运行且已被前后端依赖 | 本轮不做破坏性字段迁移；通过接口响应映射补齐规格语义，迁移另立专题 | P2 |

**不改动范围：**
- 不修改智能合约ABI与合约核心逻辑。
- 不进行数据库破坏性迁移。
- 不变更现有主流程认证与权限体系。

### 12.4 分阶段落地计划

**阶段A（P0，先达成可验收）**
- A1：统一单铸造确认Modal接入主路由资产页。
- A2：在确认Modal加入Gas估算与版税参数（复用现有能力）。
- A3：在确认Modal加入IPFS元数据JSON预览。
- A4：在资产上传区加入源码文件SHA-256计算与结果展示。

**阶段B（P1，增强一致性）**
- B1：补齐后端别名接口路径，适配规格文档命名。
- B2：上传区补齐CID回显与进度状态。

**阶段C（P2，后续优化）**
- C1：字段语义对齐专题（如 `token_id`/`nft_token_id` 统一）。
- C2：枚举语义与文档统一（不影响现网前提下分批处理）。

### 12.5 变更控制与风险防护

**变更控制：**
- 每个差距项单独提交，避免混合改动。
- 先补前端展示与交互，再补后端别名兼容。
- 每次改动保持接口向后兼容。

**风险与防护：**
- 风险R1：路由行为变化导致页面入口异常  
  防护：不调整现有路由树，仅增强页面内行为。
- 风险R2：上传流程改动影响创建资产成功率  
  防护：保留现有创建接口；新增展示逻辑不改变提交主路径。
- 风险R3：签名与Gas估算逻辑波动  
  防护：仅复用已存在服务调用，不新增链上写操作分支。

### 12.6 验收标准（最小可验收）

| 验收项 | 通过标准 |
|--------|----------|
| 单铸造确认流程 | 点击“铸造NFT”先弹确认Modal，含资产摘要、钱包地址、版税、Gas估算、确认按钮 |
| 元数据预览 | 确认Modal可见JSON预览，字段覆盖 `name/description/image/attributes/rights_declaration/original_creator/creation_timestamp` |
| 源码哈希按钮 | 可对源码文件计算SHA-256并展示结果；后端校验接口可返回匹配结果 |
| 批量铸造 | 支持多选，最大50，结果展示成功/失败/进行中 |
| 历史记录 | 能展示资产名、Token ID、时间、交易哈希、状态，哈希可跳转浏览器 |
| API兼容 | 旧路径与新增别名路径均可访问 |
