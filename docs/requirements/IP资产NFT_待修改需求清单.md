## 2. 修改目标

---

## 3. 待修改清单（按优先级）

## 3.1 P0（必须优先完成）

### P0-01 前端NFT页面去Mock并接入真实API
**修改内容**
- `/nft` 页面单铸、批铸、状态查询、重试全部接入后端接口
- 删除或停用页面层和Hook层的MOCK数据逻辑

**验收标准**
- 页面数据来源为真实接口返回
- 铸造后可看到真实 `token_id`、`tx_hash`、`status`

### P0-02 统一API路径与文档定义
**修改内容**
- 对齐IPFS与NFT相关接口命名与路由
- 统一“需求文档 / 前端调用 / 后端路由”三方路径定义

**验收标准**
- 无路径错配导致的404
- 接口文档与代码实现一致

### P0-03 修正铸造状态门槛
**修改内容**
- 铸造前置状态从当前实现调整为“审批通过后可铸造”
- 同步校正状态流转逻辑

**验收标准**
- 未审批通过资产无法进入铸造
- 审批通过资产可正常铸造

### P0-04 补齐核心数据模型字段
**修改内容**
- 资产表补充：`inventors`（多人）、`rights_declaration`
- 附件表补充：`is_primary`
- 同步更新Schema、DTO、接口入参/出参

**验收标准**
- 前后端可完整读写新增字段
- 历史数据迁移与兼容方案可执行

---

## 3.2 P1（应在本轮完成）

### P1-01 补齐资产登记表单字段与校验
**修改内容**
- 创作/发明人改为标签多人输入
- 新增权利声明字段
- 完善法律状态枚举和校验

**验收标准**
- 表单字段与规格说明逐项一致
- 必填、长度、日期等校验可用

### P1-02 增强附件上传体验
**修改内容**
- 增加上传进度条
- 图片上传前预览
- 上传完成展示CID
- 强化类型、大小、数量限制提示

**验收标准**
- 用户可见上传进度与CID结果
- 超限/非法类型有明确反馈

### P1-03 落地代码资产SHA-256流程
**修改内容**
- 前端增加“计算文件哈希”操作
- 代码类资产默认提交哈希而非源码
- 后端执行哈希比对并记录结果

**验收标准**
- 代码资产可完成哈希提交流程
- 服务端可校验客户端哈希一致性

### P1-04 完整化铸造确认Modal
**修改内容**
- 展示接收钱包地址
- 增加版税设置（接收地址、0%-10%比例）
- 集成Gas估算提示
- 接入MetaMask签名流程

**验收标准**
- 确认前可见完整参数摘要
- 签名与发起交易链路可用

### P1-05 批量铸造进度与结果细化
**修改内容**
- 展示逐条进行中/成功/失败状态
- 显示失败原因与重试入口
- 前端校验一次最多50条

**验收标准**
- 批量任务全程可追踪
- 失败项可定位与重试

### P1-06 铸造历史页改为真实数据驱动
**修改内容**
- 接入后端历史查询接口
- 交易哈希外链支持链浏览器跳转

**验收标准**
- 历史列表可分页筛选
- 数据与后端记录一致
