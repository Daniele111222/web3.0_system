## 2. 用户注册登录流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 前端
    participant API as 认证API
    participant DB as PostgreSQL
    participant Email as 邮件服务

    %% 注册流程
    rect rgb(200, 220, 240)
        Note over U,Email: 用户注册流程
        U->>FE: 填写注册表单<br/>(邮箱、密码、用户名、企业名称)
        FE->>API: POST /auth/register
        API->>API: 密码强度校验
        API->>DB: 创建用户记录
        API->>DB: 创建企业记录<br/>(用户设为OWNER)
        API->>Email: 发送邮箱验证链接
        API-->>FE: 201 Created
        FE-->>U: 显示"请查收验证邮件"
        Email->>U: 验证链接邮件
        U->>FE: 点击验证链接
        FE->>API: POST /auth/verify-email?token=xxx
        API->>DB: 标记邮箱已验证
        API-->>FE: 200 OK
    end

    %% 登录流程
    rect rgb(220, 240, 220)
        Note over U,API: 用户登录流程
        U->>FE: 输入邮箱/密码
        FE->>API: POST /auth/login
        API->>DB: 验证凭据
        API->>DB: 生成JWT令牌
        API-->>FE: 200 OK<br/>{access_token, refresh_token}
        FE->>FE: 存储令牌到localStorage
        FE->>U: 跳转至Dashboard
    end

    %% Token刷新流程
    rect rgb(240, 220, 220)
        Note over U,API: Token刷新流程
        U->>FE: 访问受保护页面
        FE->>API: GET /resource<br/>Header: Authorization: Bearer <access_token>
        API-->>FE: 401 Unauthorized<br/>(token过期)
        FE->>API: POST /auth/refresh<br/>{refresh_token}
        API->>API: 验证refresh_token
        API-->>FE: 200 OK<br/>{new_access_token}
        FE->>API: 重试原请求
        API-->>FE: 200 OK
    end
```

## 5. 审批工作流

```mermaid
stateDiagram-v2
    [*] --> DRAFT: 创建资产
    DRAFT --> PENDING: 提交审批
    PENDING --> APPROVED: 审批通过
    PENDING --> REJECTED: 审批拒绝
    REJECTED --> DRAFT: 退回修改
    REJECTED --> PENDING: 重新提交
    APPROVED --> MINTING: 触发铸造
    MINTING --> MINTED: 铸造成功
    MINTING --> MINT_FAILED: 铸造失败
    MINT_FAILED --> MINTING: 重试铸造
    MINT_FAILED --> [*]: 放弃
    MINTED --> TRANSFERRED: 所有权转移
    MINTED --> LICENSED: 对外授权
    MINTED --> STAKED: 质押
```

```mermaid
sequenceDiagram
    participant Creator as 资产创建者
    participant FE as 前端
    participant API as 资产API
    participant DB as PostgreSQL
    participant Approver as 审批人<br/>(OWNER/ADMIN)
    participant ApprovalAPI as 审批API

    Note over Creator,ApprovalAPI: 提交审批流程
    Creator->>FE: 点击"提交审批"
    FE->>API: POST /assets/{id}/submit-approval
    API->>DB: 更新资产状态<br/>DRAFT → PENDING
    API->>DB: 创建审批记录
    API-->>FE: 200 OK
    FE-->>Creator: 显示"已提交审批"

    Note over Approver,DB: 审批处理流程
    Approver->>FE: 查看待审批列表
    FE->>ApprovalAPI: GET /approvals/pending
    ApprovalAPI-->>FE: 待审批列表
    FE-->>Approver: 显示审批列表

    Approver->>FE: 点击查看详情
    FE->>ApprovalAPI: GET /approvals/{id}
    ApprovalAPI-->>FE: 审批详情+资产信息
    FE-->>Approver: 显示详情页

    Note over Approver,DB: 审批通过
    Approver->>FE: 点击"通过"
    FE->>ApprovalAPI: POST /approvals/{id}/process<br/>{action: APPROVE}
    ApprovalAPI->>DB: 更新审批状态<br/>PENDING → APPROVED
    ApprovalAPI->>DB: 更新资产状态<br/>PENDING → APPROVED
    ApprovalAPI-->>FE: 200 OK
    FE-->>Approver: 显示"审批已通过"
    FE-->>Creator: 通知:审批已通过

    Note over Approver,DB: 审批拒绝(类似)
    Approver->>FE: 点击"拒绝"并填写原因
    FE->>ApprovalAPI: POST /approvals/{id}/process<br/>{action: REJECT, reason: xxx}
    ApprovalAPI->>DB: 更新状态<br/>PENDING → REJECTED
    ApprovalAPI-->>FE: 200 OK
```

## 6. NFT铸造流程

```mermaid
sequenceDiagram
    participant U as 用户<br/>(ADMIN/OWNER)
    participant FE as 前端
    participant API as NFT API
    participant NFTService as NFT服务
    participant IPFS as IPFS服务
    participant BC as 区块链
    participant DB as PostgreSQL

    Note over U,DB: NFT铸造完整流程
    U->>FE: 选择已批准资产
    FE->>FE: 点击"铸造NFT"
    FE->>FE: 显示铸造确认Modal
    Note right of FE: 接收钱包地址<br/>版税设置(可选)<br/>Gas费预估
    U->>FE: 确认铸造
    FE->>API: POST /nft/mint<br/>{asset_id, minter_address}
    API->>NFTService: mint_asset_nft()

    Note over NFTService,IPFS: 步骤1:生成元数据
    NFTService->>NFTService: 验证资产状态=APPROVED
    NFTService->>NFTService: 验证有附件
    NFTService->>NFTService: 生成NFT元数据JSON
    Note right of NFTService: name, description<br/>image: ipfs://主图CID<br/>attributes: 类型/创作人等<br/>rights_declaration

    Note over NFTService,IPFS: 步骤2:上传元数据到IPFS
    NFTService->>IPFS: POST /ipfs/upload-json<br/>{metadata}
    IPFS-->>NFTService: {ipfs_cid}<br/>ipfs://Qmxxx

    Note over NFTService,BC: 步骤3:调用合约铸造
    NFTService->>BC: mint(owner, ipfs_uri)
    BC->>BC: 执行交易
    BC-->>NFTService: {token_id, tx_hash}

    Note over NFTService,DB: 步骤4:更新数据库
    NFTService->>DB: 更新资产状态<br/>APPROVED → MINTED
    NFTService->>DB: 记录token_id, mint_tx_hash
    NFTService->>DB: 创建MintRecord

    NFTService-->>API: {token_id, tx_hash}
    API-->>FE: 200 OK
    FE-->>U: 显示铸造成功
    U->>FE: 查看NFT详情
    FE->>API: GET /ownership/assets/{token_id}
    API-->>FE: NFT详情+元数据
```

```mermaid
graph TB
    subgraph 铸造前检查["铸造前置条件检查"]

    C1 --> C2 --> C3 --> C4
    C4 --> M1 --> M2 --> M3 --> M4 --> M5
    M5 --> B1 --> B2 --> B3 --> B4 --> B5
    B5 --> D1 --> D2 --> D3 --> D4
```

## 7. NFT所有权转移流程

```mermaid
sequenceDiagram
    participant Owner as 当前所有者
    participant FE as 前端
    participant API as 权属API
    participant OwnershipService as 权属服务
    participant BC as 区块链
    participant DB as PostgreSQL
    participant Recipient as 接收方

    Note over Owner,Recipient: NFT所有权转移完整流程
    Owner->>FE: 选择NFT
    FE->>API: GET /ownership/assets/{token_id}
    API-->>FE: NFT详情
    FE-->>Owner: 显示NFT详情

    Owner->>FE: 点击"转移"
    FE->>FE: 显示转移Modal
    Note right of FE: 目标钱包地址<br/>转移原因(可选)<br/>Gas费预估
    Owner->>FE: 输入目标地址和原因
    Owner->>FE: 确认转移
    FE->>FE: 调用MetaMask签名
    FE->>API: POST /ownership/transfer<br/>{token_id, to_address, reason}
    API->>OwnershipService: transfer_nft()

    Note over OwnershipService,BC: 验证并执行
    OwnershipService->>BC: ownerOf(token_id)
    BC-->>OwnershipService: current_owner
    OwnershipService->>OwnershipService: 验证调用者=current_owner

    OwnershipService->>BC: transferNFT(from, to, tokenId, reason)
    BC->>BC: 执行转移交易
    BC-->>OwnershipService: tx_hash

    Note over OwnershipService,DB: 更新数据库
    OwnershipService->>DB: 创建Ownership记录
    OwnershipService->>DB: 更新原记录状态

    OwnershipService-->>API: {success, tx_hash}
    API-->>FE: 200 OK
    FE-->>Owner: 显示"转移成功"

    Note over Recipient,BC: 验证接收
    BC->>BC: 触发NFTTransferred事件
    Owner->>FE: 查看区块链浏览器
    FE->>BC: 查询转移记录
    BC-->>FE: 显示最新owner
```

## 8. 钱包绑定流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 前端
    participant API as 认证API
    participant BC as 区块链

    Note over U,BC: 钱包绑定完整流程
    U->>FE: 进入设置页面
    FE->>FE: 点击"绑定钱包"
    FE->>FE: 调用window.ethereum<br/>/request({method: 'eth_requestAccounts'})
    FE-->>U: MetaMask弹窗
    U->>FE: 选择并连接钱包
    FE->>FE: 获取walletAddress

    Note over FE,API: 签名验证流程
    FE->>API: GET /auth/wallet/message<br/>{walletAddress}
    API->>API: 生成随机消息<br/>包含walletAddress和timestamp
    API-->>FE: {message, nonce}
    FE-->>U: 显示签名请求
    U->>FE: 对消息签名
    FE->>FE: 获取signature

    FE->>API: POST /auth/bind-wallet<br/>{wallet_address, signature, message, nonce}
    API->>API: 使用ecrecover验证签名
    API->>API: 验证恢复的address == wallet_address
    API->>API: 更新用户wallet_address
    API-->>FE: 200 OK
    FE-->>U: 显示"钱包绑定成功"
```

```mermaid
graph TB
    subgraph 绑定条件["钱包绑定条件"]

        C3[签名验证通过]
    end

    subgraph 签名内容["签名消息内容"]
        S1[随机nonce]
        S2[wallet_address]
        S3[timestamp]
        S4[过期时间]
    end

    subgraph 验证过程["服务端验证"]
        V1[提取message内容]
        V2[使用ecrecover恢复地址]
        V3[比较恢复地址与提交的wallet_address]
        V4[验证未过期]
    end

    C1 --> C2 --> C3
    S1 --> S2 --> S3 --> S4
    V1 --> V2 --> V3 --> V4
```

## 附录: 批量操作流程

### 批量铸造流程

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 前端
    participant API as NFT API
    participant NFTService as NFT服务
    participant BC as 区块链
    participant DB as PostgreSQL

    Note over U,DB: 批量铸造NFT(最多50个)
    U->>FE: 选择多个已批准资产<br/>(勾选)
    FE->>FE: 点击"批量铸造"
    FE->>FE: 显示批量确认Modal<br/>显示总数和Gas费预估
    U->>FE: 确认批量铸造
    FE->>API: POST /nft/batch-mint<br/>{asset_ids: [id1, id2, ...]}
    API->>NFTService: batch_mint_assets()

    Note over NFTService,BC: 批量处理
    NFTService->>NFTService: 验证总数≤50
    loop 遍历每个资产
        NFTService->>NFTService: 验证状态=APPROVED
        NFTService->>NFTService: 生成元数据
        NFTService->>BC: mint(to, metadataURI)
    end

    NFTService-->>API: 返回结果列表<br/>{success: [...], failed: [...]}
    API-->>FE: 207 Multi-Status<br/>(部分成功)
    FE-->>U: 显示铸造结果<br/>成功数/失败数
```

sequenceDiagram
